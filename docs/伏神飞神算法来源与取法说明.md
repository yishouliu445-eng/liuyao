# 伏神飞神算法来源与取法说明

## 目的

这份说明固定当前项目对 `伏神 / 飞神` 的唯一工程口径，避免实现时混用不同流派。
后续如果要调整算法来源，必须同时更新本文、规则 evidence 字段和回归测试。

## 当前采用的唯一口径

当前实现统一采用这条工程化取法：

1. 先以卦宫对应的纯卦作为基准卦。
2. 用纯卦六爻的纳甲与六亲分布，和当前盘面的明爻六亲做对照。
3. 纯卦里存在、但当前明爻六亲中缺失的那一项，按同位落成伏神。
4. 同一爻位已经显出的明爻，视为该伏神对应的飞神。

这就是当前 `FuShenResolver` 的唯一依据，不混用其他“从本宫首卦”“从世应旁取”“按断事经验直接补象”的口径。

## 适用范围

当前只覆盖路线图要求的第一批确定性能力：

- `用神不上卦时取伏神`
- `飞神压伏`
- `伏得生扶`
- `伏被冲破`

暂不覆盖：

- 更复杂的伏神转用优先级
- 多流派冲突时的兼容逻辑
- 以经验歌诀为主、难以稳定结构化的派生断法

## 工程取法步骤

### 1. 纯卦基准

项目当前按宫位选取纯卦基准：

- `乾 -> 乾为天`
- `兑 -> 兑为泽`
- `离 -> 离为火`
- `震 -> 震为雷`
- `巽 -> 巽为风`
- `坎 -> 坎为水`
- `艮 -> 艮为山`
- `坤 -> 坤为地`

纯卦的六爻纳甲与六亲由现有 `NaJiaResolver`、`LiuQinResolver` 统一计算，不单独维护第二套表。

### 2. 缺失六亲判定

对比“纯卦六爻六亲集合”和“当前盘面明爻六亲集合”：

- 如果某一六亲已经在明爻出现，则不为它再补伏神。
- 如果某一六亲未在明爻出现，则把纯卦该同位信息记为伏神。

因此，伏神首先是盘面数据，不是分析层临时猜测。

### 3. 飞神判定

当前工程口径里：

- 同位明爻就是飞神。
- 同位伏神就是被伏之神。

飞伏关系只按五行关系做第一版确定性判断：

- 飞神生伏
- 飞神克伏
- 伏神生飞
- 伏神克飞
- 飞伏同气

## 规则层的第一版判断

在 `FuShenFlyShenRule` 中，当前只编码以下确定性语义：

### 用神不上卦时取伏神

- 若明爻已有用神，不转伏神。
- 若明爻无用神，才从 `fuShenLiuQin` 中查找对应伏神。

### 飞神压伏

- 当飞神五行克伏神五行时，记为 `flyShenSuppress = true`。
- 这是第一版“飞神压伏”的唯一触发条件。

### 伏得生扶

- 月建或日辰与伏神同五行，视为有根。
- 月建或日辰五行生伏神五行，也视为有扶。

当前 evidence 字段：

- `hiddenUseGodSupported`
- `supportSources`

### 伏被冲破

- 月建地支冲伏神地支，或日辰地支冲伏神地支，即记为冲破。

当前 evidence 字段：

- `hiddenUseGodBroken`
- `breakSources`

## 边界约束

- 明爻可见用神优先，不为了“补全信息”强行转看伏神。
- 伏神只在 chart build 阶段生成一次，后续规则直接消费结构化字段。
- 若未来引入其他流派，必须先在文档中单列“替代口径”并给出迁移测试，不能直接混入现有实现。
