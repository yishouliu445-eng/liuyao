# 刘爻首页双入口与问题方向纠偏设计

日期：2026-04-16

## 1. 背景

当前首页存在三个核心问题：

1. 首页更像高质感表单，而不像六爻排卦系统。
2. `问题分类` 被放在高权重位置，且当前实现会直接参与后端意图判断，用户选错会把分析带偏。
3. 首屏默认暴露较多六爻术语与操作，对普通用户门槛偏高。

本设计目标是在不牺牲专业性的前提下，同时覆盖两类用户：

- 普通用户：知道自己想问什么，但不懂六爻术语与排爻流程。
- 熟手用户：希望自己设定六爻、动爻与起卦时间，保留可控性。

## 2. 已确认决策

- 首页入口命名采用：`问事 / 手工起卦`
- 首页默认视觉方向采用：`A2. 工具内含仪式感`
- 文案基调采用：`克制的古意`
- “问题分类”改名为：`问题方向`
- 问题方向策略采用：`文本自动识别 + 用户可改 + 冲突轻确认`

## 3. 产品目标

### 3.1 目标

- 让普通用户第一次进入时，不懂六爻术语也能完成一次起问。
- 让熟手用户能快速切换到手工排爻，不觉得系统过度简化。
- 让首页第一眼就具备明确的六爻品类识别。
- 降低因为“用户误选分类”导致的断卦偏差。

### 3.2 非目标

- 不在首页营造浓重玄学包装感。
- 不使用“智能起卦”“AI 起卦”等包装式命名。
- 不让问题方向成为强制前置步骤。

## 4. 首页信息架构

### 4.1 总体原则

- 首屏主任务只有一个：开始一次问事或起卦。
- 左侧为主工作区，右侧为辅助说明区。
- 六爻元素必须成为首屏的重要视觉锚点，但不能阻断普通用户。

### 4.2 首页入口结构

首页首屏顶部提供切换：

- `问事`
- `手工起卦`

默认进入 `问事`。

切换原则：

- `问事`：适合普通用户。先写问题，再识别方向，再进入后续问事流程。
- `手工起卦`：适合熟手用户。保留问题输入，但把六爻排盘区提升为主操作区。

### 4.3 问事模式首屏结构

顺序如下：

1. 入口切换：`问事 / 手工起卦`
2. 主标题：`先写下所问之事。`
3. 辅助说明：`问题方向会先自动识别，若与你本意不符，可手动改定；如需自行排六爻，可切换到手工起卦。`
4. 问题输入框
5. 系统识别出的 `问题方向`
6. `问事时间`
7. 主按钮：`进入问事`

右侧辅助区放以下内容：

- 信任说明：依六爻之法起问，支持多轮追问、案例留痕与应验回看
- 极淡的仪式元素说明或视觉装饰
- 历史 / 日历 / 案例入口

### 4.4 手工起卦模式首屏结构

顺序如下：

1. 入口切换：`问事 / 手工起卦`
2. 主标题：保留同一语气体系
3. 问题输入框
4. `问题方向`（可选）
5. 六爻操作区
6. `起卦时间`
7. `起卦方式`
8. 主按钮：`开始起卦`

手工起卦模式下：

- 六爻操作区成为首屏主角
- 点击切换阴阳、长按或双击设置动爻
- 不再用“进入会话”作为主动作词

## 5. 问题方向设计

### 5.1 角色重新定义

`问题方向` 不再是必须由用户先选对的强前置字段，而是三种信息的组合结果：

- `questionText`：用户真实输入的问题文本
- `userSelectedDirection`：用户手动修改后的方向，可为空
- `detectedDirection`：系统根据问题文本识别出的方向

最终真正参与分析的字段为：

- `finalDirection`

### 5.2 判定优先级

推荐优先级如下：

1. 先根据 `questionText` 自动识别 `detectedDirection`
2. 若用户未手动修改，则 `finalDirection = detectedDirection`
3. 若用户手动改了，且与系统识别一致，则 `finalDirection = userSelectedDirection`
4. 若用户手动改了，且与系统识别冲突，则提交前进行一次轻确认

### 5.3 冲突轻确认

触发条件：

- `userSelectedDirection` 与 `detectedDirection` 不一致
- 且 `detectedDirection` 置信度达到阈值

确认文案建议：

`你选择的是“出行”，但问题更像“感情”，是否按“感情”解读？`

操作项：

- `按感情解读`
- `仍按出行`

原则：

- 只在提交前轻提示一次
- 不打断正常输入流程
- 不使用技术性词汇，如“识别置信度”“分类冲突”

## 6. 后端语义调整

当前实现中，`questionCategory` 被优先用于意图解析，风险较高。建议改为以下语义拆分：

- `questionText`
- `userSelectedDirection`
- `detectedDirection`
- `finalDirection`
- `directionSource`
- `directionConfidence`

后端行为调整建议：

- 不再无条件“先信用户点选方向”
- `finalDirection` 才进入用神判定与分析文案分支
- `userSelectedDirection` 主要用于记录用户意图输入
- 历史列表、案例筛选优先展示 `finalDirection`

兼容策略：

- 若旧接口只有 `questionCategory`，短期内可将其映射为 `userSelectedDirection`
- 新逻辑生成 `finalDirection` 后再继续后续分析链路

## 7. 文案系统

### 7.1 文案风格原则

- 用词稳重、简洁、克制
- 允许有少量古意，但不文绉绉
- 不带江湖气，不带营销产品词

### 7.2 推荐用语

- `问事`
- `手工起卦`
- `先写下所问之事。`
- `若与你本意不符，可手动改定`
- `进入问事`
- `开始起卦`
- `所问何事`
- `问题方向`
- `问事时间`
- `起卦时间`

### 7.3 避免用语

- `智能起卦`
- `AI 起卦`
- `聚焦方向`
- `请输入你想占问的事`
- `进入会话`
- `洞察`
- `解锁`

## 8. 视觉方向

### 8.1 整体气质

- 基调：深色、沉静、清晰
- 气质：工具可信度优先，仪式感只作衬托
- 仪式元素：极淡的卦线、干支、纸墨纹理

避免：

- 古风人物
- 山水云雾大图
- 符咒贴图
- 过重的神秘主义视觉

### 8.2 视觉锚点

首屏必须明确出现六爻相关识别元素：

- 六爻排盘区
- 卦线或排爻结构
- 与六爻相关的极简纹理

不能只是一张高质感暗色表单。

### 8.3 选中态按钮可读性规范

已发现问题：

- `问事` 与旁侧按钮的底色和字色过于接近，导致文字发糊。

修正原则：

- 亮底必须配深字
- 暗底必须配亮字
- 避免“中亮度底 + 中亮度字”
- 选中态不要使用脏金半透明底叠金字

建议：

- 选中态：浅金或暖米金底 + 深墨色文字
- 未选中态：深底 + 浅金或偏白文字
- 字重不低于 `500`
- 降低边框抢眼程度，优先靠底色与文字明度差建立状态

## 9. 响应式布局

### 9.1 Desktop

- 左主右辅
- 左侧为问事或手工起卦主流程
- 右侧为说明、信任建立与次入口

### 9.2 Mobile

顺序压缩为：

1. 入口切换
2. 主标题与辅助说明
3. 问题输入
4. 问题方向
5. 六爻区或时间区
6. 主按钮

原则：

- 首屏不要同时堆满导航、分类标签、六爻、说明卡片
- 移动端要优先保证主按钮和问题输入易读易点

## 10. 关键交互状态

### 10.1 问事模式

- 初始：空白输入，问题方向未展示或展示为“待识别”
- 输入中：实时识别问题方向
- 用户手动改方向：记录用户选择
- 提交前冲突：弹轻确认
- 确认后：进入问事流程

### 10.2 手工起卦模式

- 初始：展示默认六爻状态
- 切换阴阳：即时反馈
- 设置动爻：有明确视觉标记
- 提交：直接进入起卦分析

## 11. 验收标准

设计完成后的验收标准：

1. 普通用户第一次进入，不懂术语也能完成一次问事。
2. 熟手用户能在 1 次切换内找到手工起卦入口。
3. 首屏一眼能识别这是六爻系统，而不是泛 AI 表单页。
4. 用户误选方向时，不会无提示地把整次分析带偏。
5. 选中态按钮在暗背景下保持清晰可读，不再出现文字发糊。
