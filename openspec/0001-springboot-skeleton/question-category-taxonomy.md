# 六爻问类体系（第一版）

## 目标

给当前六爻工程补一套可维护的问类体系，解决“只靠少数硬编码分类撑模板、规则和知识召回”的问题。

## 当前标准问类与现有 intent 的映射

- 求职 -> `JOB_OPPORTUNITY`
- 工作 -> `JOB_STABILITY`
- 收入 -> `INCOME`
- 压力 -> `PRESSURE`
- 人际 -> `RELATION`
- 成长 -> `GROWTH`
- 考试 -> `EXAM`
- 健康 -> `HEALTH`
- 出行 -> `TRAVEL`
- 感情 -> `EMOTION`
- 合作 -> `COOPERATION`

## 第一版别名示例

- 工资 / 薪资 / 奖金 / 财运 -> 收入
- 面试 / 录用 / offer / 入职 -> 求职
- 工作稳定 / 保工作 / 稳定性 -> 工作
- 关系 / 同事 / 上下级 -> 人际
- 学习 / 提升 -> 成长
- 考证 / 考核 -> 考试
- 婚姻 / 恋爱 / 复合 -> 感情
- 旅行 / 差旅 -> 出行
- 签约 / 合同 -> 合作

## 实施原则

1. 对外优先使用中文标准问类
2. 对内继续复用现有 `QuestionIntent`
3. 当前未知分类不强行改写，先保留原值
4. 后续新增类目先补文档，再补规范化和测试
