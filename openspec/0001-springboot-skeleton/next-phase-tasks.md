# 第二阶段增强项清单

## 目标

在当前“骨架 + 最小闭环 + 案例查询”基础上，继续把系统推进到更适合联调和迭代的版本。

## 推荐优先级

### P1 排盘底座继续做真

- 校正六神、纳甲、六亲的更多传统口径边界
- 增加更多固定样例回归测试
- 明确本卦/变卦/世应/卦宫的可追踪调试字段

### P1 规则层增强

- 补健康、出行、合作相关细分规则
- 增强规则 evidence 的结构一致性
- 将更多规则从启发式过渡到真实盘面字段驱动

### P1 接口与交付能力

- 增加案例详情的 controller 级异常测试
- 补更正式的 API 文档
- 增加 README 中的部署与联调说明

### P2 查询与运营能力

- 增加案例筛选、分页、按分类查询
- 增加案例标题、标签、反馈字段
- 增加规则命中统计与简单报表接口

### P2 分析层增强

- 将当前 stub 分析替换为可配置分析服务
- 引入知识库检索前的接口预留
- 规范分析 prompt 所需的结构化输入

### P3 工程化完善

- 增加 PostgreSQL 本地 docker 启动脚本
- 增加 test profile 与 dev profile
- 增加基础 CI 检查
